# Hex Edit (Binary Field Annotator)

A GTK3-based hex editor and binary field annotation tool for reverse engineers,
security researchers, and protocol analysts. It can inspect, annotate, edit, and
script binary file structures from a dark/light desktop UI.

The current primary application is `hex_edit_gtk.py`, a PyGObject/GTK3 port of
the original CustomTkinter prototype. The older Tk/CustomTkinter implementation
is kept as `hex_edit-tk.py`.

> This project was generated by AI through an iterative conversation-driven
> development process.

## Features

- **GTK3 desktop UI** - Uses PyGObject, `Gtk.Paned`, `Gtk.Notebook`,
  `Gtk.TextView`, `Gtk.TextBuffer`, and `Gtk.TextTag`.
- **Multi-file tabs** - Open multiple binaries in one window; pristine tabs are
  reused and modified tabs show a marker.
- **Hex + ASCII view** - Configurable bytes per row (8 / 16 / 32), centered byte
  cells, scrollable output, hover status, and aligned ASCII pane.
- **Field annotation** - Select byte ranges and color-code them as named fields
  with notes.
- **Editable fields** - Click a field in the table to edit its name, offsets,
  color, or note without deleting and recreating it.
- **Custom color memory** - Custom colors are tracked, usage-sorted, capped to a
  stable swatch strip, and saved with annotations.
- **Inline hex editing** - Toggle Hex Edit mode, click a byte, and type hex
  digits to overwrite bytes in place.
- **Right-click byte editor** - Dialog-based single-byte editor with synchronized
  hex and decimal fields.
- **Live inspector** - Decode selections or fields as integers, floats, strings,
  hex, binary, or decimal bytes with endian and signedness controls.
- **Embedded Python console** - Run trusted local Python inside the app, import
  modules, and inspect or mutate the active tab's buffer through `buf` and
  helper functions.
- **Dockable / floating console** - Keep the Python console docked as a bottom
  pane, pop it out into its own GTK window, or drag it back over the main window
  to dock it again.
- **Theme and scaling** - Dark/light theme support plus keyboard scaling across
  all open tabs.
- **Annotation persistence** - Save and load `.fields.json` annotation files.
- **AI annotation template** - Includes prompt/template files for asking an AI
  model to generate loadable protocol annotations.

### GTK version (primary)

| Dependency | Notes |
|---|---|
| Python 3.10+ | Uses modern union type syntax such as `bytearray | None` |
| GTK3 | Usually present on Kali/Xfce and many Linux desktops |

Kali / Debian / Ubuntu:

```bash
sudo apt install python3-gi python3-gi-cairo gir1.2-gtk-3.0
```

The GTK version is intended primarily for Linux/Kali. Running PyGObject on
Windows is possible but not the primary target for this port.

### Legacy Tk version

`hex_edit-tk.py` uses CustomTkinter:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install customtkinter
```

On Linux, it also needs tkinter, commonly installed as `python3-tk`.

## Usage

### Opening binaries and tabs

Click **Open**, or press `Ctrl+O`, and select a file. If the active tab is still
empty, the file opens there; otherwise it opens in a new tab. Use the `+` tab
button or `Ctrl+T` to create a new empty tab. Use `Ctrl+W`, the tab close button,
or middle-click a tab to close it.

### Annotating fields

1. Drag across bytes in the hex view to select a range, or use `Shift+Click` to
   extend the current selection.
2. Fill in **Name**, **Start**, **End**, and optionally **Note**.
3. Pick a palette color, or click the custom color button to choose a new color.
4. Press **Add Field** or `Enter`.

Fields appear highlighted in the hex view and listed in the **Defined Fields**
table. Double-click a table row to jump to that field in the hex view.

### Editing existing fields

Click a row in the **Defined Fields** table. Its values load into the field form.
Change the name, start/end offsets, note, or color, then press **Update
Selected** or `Enter`. Press **New Field** to clear the form and return to
creating a new annotation.

### Live byte inspection

Select bytes with the mouse, or click a field in the table, to inspect that
range. The inspector can decode bytes as:

- Integer, signed or unsigned, little or big endian
- Float16, float32, or float64
- ASCII, UTF-8, UTF-16, or Latin-1 strings
- Hex, binary, or decimal bytes

### Inline hex editing

1. Toggle **Hex Edit** in the toolbar.
2. Click any byte in the hex view to position the cursor.
3. Type two hex digits to overwrite that byte.
4. Use arrow keys to navigate.
5. Press `Esc` to exit the edit cursor.

Modified bytes are highlighted in scarlet (`#ff1744`). Right-click any byte for
the dialog-based single-byte editor.

### Embedded Python console

Click **Python** in a tab toolbar to open that tab's console as a bottom pane.
The console is trusted local Python running in the app process. Imports and
filesystem access are allowed, and long-running code can block the UI.

Each tab has its own console state and namespace.

Available console names:

| Name | Purpose |
|---|---|
| `buf` | Live `bytearray` for the current tab's binary |
| `app` | The tab's `BinaryFieldAnnotator` instance |
| `refresh()` | Re-render the hex view and inspector |
| `write(offset, data)` | Overwrite bytes from an int, bytes, bytearray, or iterable of ints |
| `insert(offset, data)` | Insert bytes into the buffer |
| `delete(start, end=None)` | Delete one byte or an inclusive byte range |
| `hexdump(start=0, length=256)` | Return a formatted text dump |
| `len_buf()` | Return the current buffer length |

Examples:

```python
len_buf()
hexdump(0, 64)
write(0, b"ABCD")
buf[4] = 0xFF
insert(8, bytes([0x00, 0x01]))
delete(0x10, 0x1F)
refresh()
```

The hex pane refreshes after buffer changes. Helper edits are tracked
efficiently; direct `buf` mutations are detected after command execution.

Use **Pop Out** to move the console into its own window. Use **Dock**, or drag
the floating console back over the main app window and release, to snap it back
into the tab's bottom pane.

### Saving

| Action | How |
|---|---|
| Save annotations | **Save Annot.** button or `Ctrl+S` |
| Load annotations | **Load Annot.** button or `Ctrl+L` |
| Save edited binary | **Save Binary As...** button or `Ctrl+Shift+S` |

## Annotation JSON Format

Annotations are saved as plain JSON and can be edited by hand or
version-controlled:

```json
{
  "filepath": "/path/to/original.bin",
  "fields": [
    {
      "id": 0,
      "name": "Magic",
      "start": 0,
      "end": 3,
      "color": "#ff6b6b",
      "note": "ELF magic bytes"
    }
  ],
  "custom_colors": [
    "#123456"
  ]
}
```

`start` and `end` are inclusive byte offsets. `custom_colors` is optional and is
used to restore custom color swatches when loading annotation files.

## Architecture Notes

- **Primary app** - `hex_edit_gtk.py`, a GTK3/PyGObject application.
- **Legacy app** - `hex_edit-tk.py`, the earlier CustomTkinter implementation.
- **Controller** - `MultiFileApp` owns the shared GTK window, global theme/zoom,
  notebook tabs, and global keyboard shortcuts.
- **Tab state** - Each `BinaryFieldAnnotator` tab owns its binary buffer,
  annotations, panes, inspector, Python console, and save state.
- **Hex view** - `Gtk.TextView` + `Gtk.TextBuffer` + `Gtk.TextTag`, mirroring the
  original Tk tag layering model.
- **Fields table** - `Gtk.TreeView` backed by a `Gtk.ListStore`.
- **Embedded console** - Python `code.InteractiveConsole` with captured
  stdout/stderr and a persistent namespace per tab.
- **Styling** - A shared GTK CSS provider drives theme colors, font sizing, and
  widget styling across tabs.
