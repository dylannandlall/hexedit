# Hex Edit (Binary Field Annotator)

A modern hex editor and binary field annotation tool with a dark/light
CustomTkinter UI. It is designed for reverse engineers, security researchers,
and protocol analysts who need to inspect, annotate, script, and edit binary
file structures.

> This project was generated by AI (Claude, Anthropic) through an iterative
> conversation-driven development process.

---

## Features

- **Hex + ASCII View** - Full binary display with configurable bytes per row
  (8 / 16 / 32). Each byte is rendered in a centered cell for cleaner
  annotation highlighting.
- **Field Annotation** - Select byte ranges and color-code them as named fields
  with notes.
- **Editable Fields** - Click a field in the table to edit its name, offsets,
  color, or note without deleting and recreating it.
- **Custom Color Memory** - Colors chosen with the custom color picker are added
  as reusable swatches and saved with annotation files.
- **Inline Hex Editing** - Toggle Hex Edit mode, click a byte, and type hex
  digits to overwrite bytes in place.
- **Right-click Byte Editor** - Dialog-based single-byte editor with synchronized
  hex and decimal fields.
- **Live Inspector** - Decode selections or fields as integers, floats, strings,
  hex, binary, or decimal bytes with endian and signedness controls.
- **Embedded Python Console** - Run trusted local Python inside the app, import
  modules, and inspect or mutate the active buffer through `buf` and helper
  functions.
- **Dockable / Floating Console** - Keep the Python console docked as a bottom
  pane, pop it out into its own window, or drag it back over the main window to
  dock it again.
- **Smooth Pane Resizing** - Pane drags use non-opaque proxy resize behavior for
  smoother interaction on Windows while remaining portable to Linux/Kali.
- **Application Scaling** - Use keyboard shortcuts to scale the UI larger or
  smaller.
- **Annotation Persistence** - Save and load field annotations as JSON.
- **AI Annotation Template** - Includes a reusable prompt template for asking an
  AI model to generate loadable protocol annotations.

## Installation

### 1. Clone or download

```bash
git clone <repo-url>
cd <repo-directory>
```

### 2. Create a virtual environment

Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install customtkinter
```

Linux / Kali / Debian:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install customtkinter
```

### 3. Ensure tkinter is installed on Linux

```bash
# Debian / Ubuntu / Kali
sudo apt install python3-tk

# Fedora / RHEL
sudo dnf install python3-tkinter

# Arch
sudo pacman -S tk
```

### 4. Run

Windows PowerShell:

```powershell
.\.venv\Scripts\python hex_edit.py
```

Linux / Kali / Debian:

```bash
source .venv/bin/activate
python3 hex_edit.py
```

## Usage

### Opening a binary

Click **Open** in the toolbar, or press `Ctrl+O`, and select any file. The hex
view populates immediately.

### Annotating fields

1. Drag across bytes in the hex view to select a range, or use `Shift+Click` to
   extend the current selection.
2. Fill in **Name**, **Start**, **End**, and optionally **Note**.
3. Pick a palette color, or click the custom color button to choose a new color.
4. Press **Add Field** or `Enter`.

Fields appear highlighted in the hex view and listed in the **Defined Fields**
table. Double-click a table row to jump to that field in the hex view.

### Editing existing fields

Click a row in the **Defined Fields** table. Its values load into the field form.
Change the name, start/end offsets, note, or color, then press **Update
Selected** or `Enter`. Press **New Field** to clear the form and return to
creating a new annotation.

### Live byte inspection

Select bytes with the mouse, or click a field in the table, to inspect that
range. The inspector can decode bytes as:

- Integer, signed or unsigned, little or big endian
- Float16, float32, or float64
- ASCII, UTF-8, UTF-16, or Latin-1 strings
- Hex, binary, or decimal bytes

### Inline hex editing

1. Toggle **Hex Edit** in the toolbar.
2. Click any byte in the hex view to position the cursor.
3. Type two hex digits to overwrite that byte.
4. Use arrow keys to navigate.
5. Press `Esc` to exit the edit cursor.

Modified bytes are highlighted in scarlet (`#ff1744`). Right-click any byte for
the dialog-based single-byte editor.

### Embedded Python console

Click **Python** in the toolbar to open the console as a bottom pane. The console
is trusted local Python running in the app process. Imports and filesystem access
are allowed, and long-running code can block the UI.

Available console names:

| Name | Purpose |
|---|---|
| `buf` | Live `bytearray` for the currently opened binary |
| `app` | The `BinaryFieldAnnotator` instance |
| `refresh()` | Re-render the hex view and inspector |
| `write(offset, data)` | Overwrite bytes from an int, bytes, bytearray, or iterable of ints |
| `insert(offset, data)` | Insert bytes into the buffer |
| `delete(start, end=None)` | Delete one byte or an inclusive byte range |
| `hexdump(start=0, length=256)` | Return a formatted text dump |
| `len_buf()` | Return the current buffer length |

Examples:

```python
len_buf()
hexdump(0, 64)
write(0, b"ABCD")
buf[4] = 0xFF
insert(8, bytes([0x00, 0x01]))
delete(0x10, 0x1F)
refresh()
```

The hex pane refreshes after buffer changes. Helper edits are tracked
efficiently; direct `buf` mutations are detected after command execution.

Use **Pop Out** to move the console into its own window. Use **Dock**, or drag
the floating console back over the main app window and release, to snap it back
into the bottom pane.

## Annotation JSON Format

Annotations are saved as plain JSON and can be edited by hand or
version-controlled:

```json
{
  "filepath": "/path/to/original.bin",
  "fields": [
    {
      "id": 0,
      "name": "Magic",
      "start": 0,
      "end": 3,
      "color": "#ff6b6b",
      "note": "ELF magic bytes"
    }
  ],
  "custom_colors": [
    "#123456"
  ]
}
```

`start` and `end` are inclusive byte offsets. `custom_colors` is optional and is
used to restore custom color swatches when loading annotation files.

## Architecture Notes

The program is a single self-contained Python file (`hex_edit.py`).

- **UI chrome** - CustomTkinter widgets for frames, buttons, entries, switches,
  segmented controls, and the console shell.
- **Hex view, inspector, console output** - Native `tk.Text` widgets so byte
  coordinate math, tags, and efficient text updates remain available.
- **Fields table** - Native `ttk.Treeview`, themed to match the active palette.
- **Pane layout** - Native `tk.PanedWindow` with proxy drag resizing for smoother
  sash movement.
- **Embedded console** - Python `code.InteractiveConsole` with captured
  stdout/stderr and a persistent namespace.
- **Theme and scaling** - CTk widgets use light/dark token tuples; native Tk
  widgets are rethemed and resized manually.
