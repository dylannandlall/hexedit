# Hex Edit (Binary Field Annotator)

A modern, feature-rich hex editor and binary field annotation tool with a sleek dark UI. Designed for reverse engineers, security researchers, and protocol analysts who need to visually map out, annotate, and edit binary file structures.

> **This project was generated by AI (Claude, Anthropic) through an iterative conversation-driven development process.**

---

## Features

- **Hex View** — Full hex + ASCII display with configurable bytes-per-row (8 / 16 / 32)
- **Field Annotation** — Select byte ranges and color-code them as named fields, with notes
- **Inline Hex Editing** — Click into the hex view and type to overwrite bytes in replace mode (like a hex editor), with instant feedback and scarlet highlighting of modified bytes
- **Right-click Byte Editor** — Dialog-based single-byte editor with synchronized hex/decimal fields
- **Live Inspector** — Select any bytes with your mouse to decode them on the fly as:
  - Integer (any width, signed/unsigned, little/big endian)
  - Float (float16 / float32 / float64)
  - String (ASCII, UTF-8, UTF-16, Latin-1)
  - Hex, Binary, Bytes (decimal)
- **Drag-to-Select** — Click and drag across the hex view to set a field's start and end offsets; Shift+Click extends the selection
- **Save Binary As** — Export the (potentially edited) binary to a new file
- **Annotation Persistence** — Save and load field annotations as JSON
- **Modern UI** — CustomTkinter-based dark/light theme with rounded controls, grouped toolbar, live status bar, and font auto-detection

## Installation

### 1. Clone or download

```bash
git clone <repo-url>
cd <repo-directory>
# or just download hexedit.py directly
```

### 2. Install CustomTkinter

**Option A — virtual environment (recommended on Kali/Debian due to PEP 668):**

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install customtkinter
```

**Option B — user install (Kali/Debian PEP 668 override):**

```bash
pip install --user --break-system-packages customtkinter
```

**Option C — system pip (other distros / non-PEP-668 systems):**

```bash
pip install customtkinter
```

### 3. Ensure python3-tk is installed

```bash
# Debian / Ubuntu / Kali
sudo apt install python3-tk

# Fedora / RHEL
sudo dnf install python3-tkinter

# Arch
sudo pacman -S tk
```

### 4. Run

```bash
# If using a venv:
source .venv/bin/activate
python3 hexedit.py

# Otherwise:
python3 hexedit.py
```

## Usage

### Opening a binary

Click **Open** in the toolbar (or `Ctrl+O`) and select any file. The hex view populates immediately.

### Annotating fields

1. **Drag** across bytes in the hex view to select a range (or Shift+Click to extend)
2. Fill in a **Name** (and optionally a note) in the Add Field panel on the right
3. Pick a **color** from the palette (or click `…` for a custom color)
4. Press **Add Field** or `Enter`

Fields appear highlighted in the hex view and listed in the Defined Fields table. Double-click a table row to jump to it in the hex view.

### Live byte inspection

Select any bytes with the mouse — the **Inspector** pane on the right decodes them immediately. Use the format picker, endianness toggle (LE/BE), and Signed switch to change the interpretation without needing to create a field first.

Clicking a row in the Defined Fields table also shows that field's bytes in the inspector.

### Inline hex editing

1. Click **Hex Edit** in the toolbar to toggle edit mode (button turns red)
2. Click any byte in the hex view to position the cursor (highlighted white)
3. Type two hex digits to overwrite that byte — the cursor auto-advances
4. Navigate with arrow keys; `Esc` exits the edit cursor
5. Modified bytes are highlighted in scarlet (`#ff1744`)

Right-click any byte for the dialog-based single-byte editor (also works outside Hex Edit mode).

### Saving

| Action | How |
|---|---|
| Save annotations | **Save Annot.** button or `Ctrl+S` → produces a `.fields.json` file |
| Load annotations | **Load Annot.** button or `Ctrl+L` |
| Save edited binary | **Save Binary As…** button or `Ctrl+Shift+S` → prompts for output path |

### Keyboard shortcuts

| Shortcut | Action |
|---|---|
| `Ctrl+O` | Open binary file |
| `Ctrl+S` | Save annotations |
| `Ctrl+L` | Load annotations |
| `Ctrl+Shift+S` | Save binary as… |
| `Enter` | Add field |
| `Delete` | Delete selected field |
| Arrow keys | Navigate edit cursor (in Hex Edit mode) |
| `Shift+Click` | Extend current byte selection |
| `Esc` | Exit edit cursor |

```json
      "name": "Magic",

      "end": 3,
      "color": "#ff6b6b",
      "note": "ELF magic bytes"
    }
  ]
}
```

## Architecture notes

The program is a single self-contained Python file (`hexedit.py`, ~1100 lines).

- **UI chrome** — CustomTkinter (`CTkFrame`, `CTkButton`, `CTkOptionMenu`, etc.)
- **Hex view + detail pane** — native `tk.Text` with per-byte tag coloring; kept native so that `replace()`, `tag_*`, and character-column index math (`@x,y`) remain available verbatim
- **Fields table** — native `ttk.Treeview`, themed to match the CTk palette
- **Theme system** — `THEME` dict of `{dark, light}` token dicts; CTk widgets receive `(light, dark)` tuples for automatic mode-switching; native widgets are re-themed via `_retheme_native()` on appearance change
